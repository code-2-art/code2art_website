# 网站内容架构（信息结构 / IA）

## A. 顶层导航（建议 6 个主入口）

1.  **Home（十周年主入口）**
    *   十周年宣言入口（2026 活动年）
    *   最新更新（最近发布的 Archive / 公告 / 活动）
    *   三条主线：Archive / Lab / Project（Project 先占位也行）

2.  **Archive 文档库**（核心：可被检索、可被引用）
    *   **Topics / 主题**（按知识域聚合：AI影像/生成艺术/交互/工具链/理论…）
    *   **Series / 系列**（训练营课程系列、讲座系列、年度系列）
    *   **Notes / 文稿**（讲义、备忘、教程、方法论）
    *   **Resources / 资源**（工具清单、素材、链接、参考书/论文）
    *   **Glossary / 术语表**（对 SEO 和 LLM 很友好）
    *   **Timeline / 年表**（2016–2026：关键节点与内容索引）

3.  **Lab（实践入口：但仍然“文档化”）**
    *   Lab 的定位说明
    *   项目列表（每个项目一个页面：概述/过程/产出/参与方式）
    *   参与方式（开放任务、招募、协作规范、贡献指南）

4.  **Project / 展览与作品**（十周年作品展/线上馆）
    *   线上展厅（轻量：作品索引页 + 作品详情）
    *   展览计划（年内节点 / 年末大展占位）
    *   艺术家/成员页（可选）

5.  **News / 公告**
    *   官宣、月度进展、活动通知（时间线型内容，利于持续收录）

6.  **About**
    *   实验编程介绍（简洁且可引用）
    *   合作/赞助（可选）
    *   联系方式 / 社群入口

---

## B. Archive 的“内容模型”（你在飞书里怎么组织更顺）

建议把 Archive 内容统一成几类，保证长期可扩展：

*   **Article（文章/教程）**：有明确标题、摘要、正文、封面（可选）
*   **Note（短笔记/快讯）**：更短、更频繁，利于更新节奏
*   **Resource（资源卡片）**：工具/链接/书/论文/数据集
*   **ProjectDoc（项目文档）**：实验记录、过程、复盘
*   **GlossaryTerm（术语）**：词条 + 定义 + 相关链接
*   **Person / Org（人物/组织，可选）**

每条内容都建议强制具备这些字段（为了 SEO/LLM）：

*   `title`
*   `summary`（1–2 句）
*   `body`
*   `publishDate` / `updatedDate`
*   `tags`（可多级）/ `topic`（主分类）
*   `canonicalSlug`（稳定 URL）
*   `language`（如有中英双语）
*   `references`（外部链接/引用，提升可信度）
