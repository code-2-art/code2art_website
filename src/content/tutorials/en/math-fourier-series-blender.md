For richer visualization effects beyond just plotting formula curves, Jupyter has some limitations.</p><p>I needed a Python runtime environment that doesn't require worrying about graphics rendering internals while still having powerful and robust rendering capabilities.
Blender and TouchDesigner are two software options that fit this criteria.</p><p><strong>Blender</strong></p><p><img loading="lazy" alt="blender-logo" src="https://avantcontra.github.io/assets/images/blender_logo_socket-0813c3a1b4b040c0de9c30a50afd66a3.png" width="1800" height="550" class="img_ev3q"></p><p>Blender is an open-source 3D content creation software, comparable to 3D Max, Maya, etc. It's compact yet powerful, cross-platform. See its official website for details:
<a href="https://www.blender.org/" target="_blank" rel="noopener noreferrer">https://www.blender.org/</a></p><p><img loading="lazy" alt="blender-ui" src="https://avantcontra.github.io/assets/images/blender-vp200500-86dae1c03f696f08491ebaab814c3d52.png" width="1000" height="400" class="img_ev3q"></p><p>Blender's ecosystem is built on Python — plugins, extensions, and advanced customizations are all written in Python. (It can actually make real-time interactive games too, similar to Unity in that regard, though Blender focuses more on 3D content creation.)</p><br><p><strong>TouchDesigner</strong></p><p><img loading="lazy" alt="TouchDesigner" src="https://avantcontra.github.io/assets/images/touchdesigner-7d3383147d583a76c5a175c7b61dd8a0.jpg" width="1024" height="611" class="img_ev3q"></p><p>TouchDesigner is a visual programming tool, comparable to other node-based tools like <a href="https://cycling74.com/" target="_blank" rel="noopener noreferrer">MaxMSP</a> and <a href="https://vvvv.org/" target="_blank" rel="noopener noreferrer">vvvv</a>.
TouchDesigner's scripting language is also Python.</p><br><p>For me, MaxMSP covers my node-based programming needs for now. I'll check out TouchDesigner when it gets hot.</p><p>So this time I'm <strong>using Blender as the Python programming playground</strong>.</p><hr><h3 class="anchor anchorWithStickyNavbar_LWe7" id="hello-world">Hello World</h3><p>First, download Blender 2.80 from the official website. Mac, Windows, and Linux are all supported.</p><p>Starting from version 2.80, Blender had a massive update. How massive? I'm not sure either — the last time I used Blender was over a decade ago with version 2.4x, when I tried parametric architecture generation with Python. Anyway, I'm treating 2.80 as new software.</p><p>After installing 2.80:</p><p><img loading="lazy" alt="blender-index" src="https://avantcontra.github.io/assets/images/blender-index-accdbf059d86693e4699ca4dfdb7c89e.png" width="2988" height="1868" class="img_ev3q"></p><p>It has tons of features — modeling, texturing, animation, rendering, etc. I still don't know how to use most of them. Check the official tutorials.</p><p>Let's go straight to <strong>Scripting</strong> to run Python.</p><p><img loading="lazy" alt="blender-scripting" src="https://avantcontra.github.io/assets/images/blender-script-0-2700c62787541dd13552c20dc1df6fdf.png" width="2988" height="1868" class="img_ev3q"></p><p>In Scripting there's a Console — ready to use out of the box. Just type Python code directly. The classic hello world:</p><p><img loading="lazy" alt="blender-hello" src="https://avantcontra.github.io/assets/images/blender-hello-7e697fd4d69e2c5ea57ddc13cacfff1e.png" width="1938" height="1264" class="img_ev3q"></p><p>Writing only in the Console is inconvenient. Click on Text — it's actually a code text editor. Click New to start:</p><p><img loading="lazy" alt="blender-scripting" src="https://avantcontra.github.io/assets/images/blender-script-509505914bdd65b7c630385735aa6711.png" width="2994" height="1868" class="img_ev3q"></p><p>Try hello world again:</p><p><code>print ("hello world from Text Editor")</code></p><p><img loading="lazy" alt="blender-hello-from-text-0" src="https://avantcontra.github.io/assets/images/blender-hello-text-0-fbff0bafc647b846a36ffc6eaff58559.png" width="1550" height="1378" class="img_ev3q"></p><p>After entering code and clicking <strong>Run Script</strong>, you'll notice the built-in Console on the left doesn't show "hello world from Text Editor."</p><p>To debug Text Editor code, you need to open the OS system Console. Here are the official tips:
<a href="https://docs.blender.org/api/current/info_tips_and_tricks.html#use-the-terminal" target="_blank" rel="noopener noreferrer">https://docs.blender.org/api/current/info_tips_and_tricks.html#use-the-terminal</a></p><p>For Blender on Windows, simply enable the system Console from the Help menu.</p><p>For Mac, first open Terminal (close Blender if already open), then launch Blender from Terminal:</p><ul><li>Open Terminal</li><li>Right-click Blender(.app) in Applications, select "Show Package Contents"</li><li>Find Contents->MacOS->Blender</li><li>Drag 'Blender' into the Terminal window</li><li>Press Enter</li></ul><p><img loading="lazy" alt="blender-terminal-drag" src="https://avantcontra.github.io/assets/images/blender-terminal-drag-71147c04176c1fe4861236362d559a3a.png" width="1262" height="1330" class="img_ev3q"></p><p>With the system Terminal open, re-enter <code>print ("hello world from Text Editor")</code> in Blender Scripting's Text, then <strong>Run Script</strong> — you'll see the output in Terminal:</p><p><img loading="lazy" alt="blender-terminal-print" src="https://avantcontra.github.io/assets/images/blender-terminal-print-cb661fc806e332d81749fd9948b68eeb.png" width="1140" height="904" class="img_ev3q"></p><hr><h3 class="anchor anchorWithStickyNavbar_LWe7" id="grease-pencil">Grease Pencil</h3><p><strong>Grease Pencil</strong> is a new feature in Blender 2.80 — a special Blender object that lets you draw 2D graphics in 3D space, create traditional 2D animations, cutouts, motion effects, and even storyboards.</p><p><img loading="lazy" src="https://docs.blender.org/manual/en/latest/_images/grease-pencil_introduction_example.png" alt="Grease Pencil" class="img_ev3q">
